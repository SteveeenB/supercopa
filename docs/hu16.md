# HU16 — Registrar retiro o descalificación de equipo

> **Actor:** Administrador
> **Prioridad:** Alta (arrastre Sprint 1)
> **Dependencias:** MS5 (notificaciones, diferible)

---

## Escenario principal

1. El administrador accede a la gestión de equipos del torneo y selecciona "Descalificar equipo".
2. Ingresa la razón de la descalificación.
3. El sistema identifica todos los partidos pendientes del equipo en el fixture.
4. Para cada partido pendiente, el sistema marca al rival con estado `DESCANSO` (no cuenta como victoria ni W.O.).
5. Los partidos ya jugados contra el equipo retirado conservan sus resultados.
6. La acción queda registrada con fecha, hora y usuario que la ejecutó.

---

## Criterios de aceptación

- [ ] Al descalificar un equipo, sus partidos pendientes se marcan como `DESCANSO`; no suman puntos al rival.
- [ ] Los resultados de partidos ya jugados contra el equipo descalificado se mantienen válidos.
- [ ] El equipo descalificado desaparece de la tabla de posiciones activa pero se conserva en el historial del torneo.
- [ ] La acción queda registrada con fecha, hora y usuario que la ejecutó.
- [ ] Un equipo descalificado **no** puede volver a integrarse (no hay botón HABILITAR).
- [ ] Un equipo `RECHAZADO` muestra botón `HABILITAR` (amarillo) en vez de `APROBAR`.

### 6. `TorneoAdminService.java` — dos nuevos métodos

```java
@Transactional
public InscripcionDTO expulsarEquipo(UUID torneoId, UUID equipoTorneoId,
                                      String motivo, String adminCedula) {
    var et = cargarInscripcion(torneoId, equipoTorneoId);

    if (et.getEstadoInscripcion() == EstadoInscripcion.EXPULSADO) {
        return toInscripcionDTO(et); // idempotente
    }

    // 1. Marcar como EXPULSADO
    et.setEstadoInscripcion(EstadoInscripcion.EXPULSADO);
    et.setExpulsadoPor(adminCedula);
    et.setFechaExpulsion(LocalDateTime.now());
    et.setMotivoExpulsion(motivo.trim());
    equipoTorneoRepo.save(et);

    // 2. Buscar partidos pendientes del equipo
    var pendientes = partidoRepo
        .findPendientesByEquipoTorneo(torneoId, equipoTorneoId, EstadoPartido.PROGRAMADO);

    for (var p : pendientes) {
        p.setEstado(EstadoPartido.DESCANSO);
        partidoRepo.save(p);
    }

    // 3. (Opcional) Emitir evento para MS5 notificar al delegado
    // eventPublisher.publishEvent(...)

    return toInscripcionDTO(et);
}

@Transactional
public InscripcionDTO habilitarEquipo(UUID torneoId, UUID equipoTorneoId) {
    var et = cargarInscripcion(torneoId, equipoTorneoId);

    if (et.getEstadoInscripcion() != EstadoInscripcion.RECHAZADO) {
        throw new RuntimeException("Solo se puede habilitar un equipo en estado RECHAZADO.");
    }

    et.setEstadoInscripcion(EstadoInscripcion.PENDIENTE_PAGO);
    et.setMotivoRechazo(null);
    equipoTorneoRepo.save(et);

    return toInscripcionDTO(et);
}
```

### 7. `AdminTorneoController.java` — nuevos endpoints

```java
@PostMapping("/{torneoId}/inscripciones/{equipoTorneoId}/expulsar")
public ResponseEntity<InscripcionDTO> expulsar(
        @PathVariable UUID torneoId,
        @PathVariable UUID equipoTorneoId,
        @RequestBody RechazoRequestDTO req,
        @AuthenticationPrincipal Jwt jwt) {
    String admin = jwt.getClaimAsString("cedula");
    return ResponseEntity.ok(
        torneoAdminService.expulsarEquipo(torneoId, equipoTorneoId, req.getMotivo(), admin));
}

@PostMapping("/{torneoId}/inscripciones/{equipoTorneoId}/habilitar")
public ResponseEntity<InscripcionDTO> habilitar(
        @PathVariable UUID torneoId,
        @PathVariable UUID equipoTorneoId) {
    return ResponseEntity.ok(
        torneoAdminService.habilitarEquipo(torneoId, equipoTorneoId));
}
```

## Cambios en frontend

### `GestionarEquiposView.jsx`

La lógica de botones cambia según el estado:

| Estado actual | Botón 1 | Botón 2 |
|---------------|---------|---------|
| `PENDIENTE_PAGO` | APROBAR | RECHAZAR |
| `APROBADO` | *(ninguno)* | *(ninguno)* |
| `RECHAZADO` | **HABILITAR (amarillo)** | *(ninguno)* |
| `EXPULSADO` | *(ninguno — no puede volver)* | *(ninguno)* |

```jsx
// Lógica de botones (reemplaza la actual)
const actions = () => {
  switch (insc.estadoInscripcion) {
    case 'PENDIENTE_PAGO':
      return (
        <>
          <button className="action-button approve" onClick={() => handleAprobar(insc)}>
            Aprobar
          </button>
          <button className="action-button reject" onClick={() => openRechazo(insc)}>
            Rechazar
          </button>
        </>
      );
    case 'RECHAZADO':
      return (
        <button className="action-button warning" onClick={() => handleHabilitar(insc)}>
          Habilitar
        </button>
      );
    case 'EXPULSADO':
      return (
        <span className="status-pill estado-expulsado">Expulsado</span>
      );
    default:
      return null; // APROBADO
  }
};
```

Además, para los equipos `APROBADO` o `RECHAZADO` se debe agregar el botón rojo **EXPULSAR**:

### Modal de expulsión (similar al de rechazo)

```jsx
const [expulsion, setExpulsion] = useState({
  open: false, eqtId: null, equipoNombre: '', motivo: ''
});

const confirmarExpulsion = async () => {
  const insc = inscripciones.find(i => i.id === expulsion.eqtId);
  if (!insc) return;
  if (!expulsion.motivo.trim()) { setError('Indica un motivo.'); return; }
  setActioningId(insc.id);
  try {
    await expulsarEquipo(insc.torneoId, insc.id, expulsion.motivo.trim(), getToken());
    setFeedback(`"${insc.equipoNombre}" expulsado del torneo.`);
    setExpulsion({ open: false, eqtId: null, equipoNombre: '', motivo: '' });
    await loadInscripciones();
  } catch (err) {
    setError(err?.message);
  } finally {
    setActioningId(null);
  }
};
```

## Reglas de negocio a validar

- Un equipo `EXPULSADO` **no** puede volver a inscribirse en ningún torneo (validar en `DelegadoService` al inscribir).
- Si un equipo es expulsado, sus jugadores quedan liberados (pueden unirse a otro equipo en otro torneo).
- `DESCANSO` no es lo mismo que `WO` — el rival no suma puntos, goles, ni se registra como victoria.
- La tabla de posiciones debe ignorar partidos con estado `DESCANSO`.
