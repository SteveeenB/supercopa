# HU44 — Delegado gestiona plantel directamente

Como delegado
Quiero agregar jugadores directamente a mi plantel validando contra el registro oficial, asignarles número de camiseta y eliminarlos del equipo cuando sea necesario sin borrar sus estadísticas
Para tener control total del plantel


> **Actor:** Delegado
> **Prioridad:** Alta (Sprint 2 - parcial, ~3 h restantes)
> **Estado actual:** Pendiente: agregar jugador directo, asignar/editar número de camiseta, eliminar jugador

---

## Escenario principal

1. El delegado accede al módulo de gestión del plantel y selecciona **Agregar jugador**.
2. Ingresa la **cédula**; el sistema consulta el registro (MS1) y autocompleta nombre, semestre y rol.
3. Si la cédula **no existe**, el sistema informa al delegado y sugiere que el jugador solicite validación al administrador.
4. El delegado confirma la adición, asigna **opcionalmente** un número de camiseta y el jugador queda en el plantel.
5. Para **eliminar** un jugador, el delegado lo selecciona y confirma la acción; si el torneo ya inició (`EN_CURSO`), el sistema advierte del impacto.
6. Las estadísticas del jugador eliminado se conservan en su perfil personal y en el historial del torneo.
7. El número de camiseta es **asignable y editable** por el delegado en cualquier momento.

---

## Criterios de aceptación

- [ ] Solo se pueden agregar jugadores cuya cédula exista en el registro (MS1).
- [ ] Un jugador **no** puede estar en dos equipos del mismo torneo; el sistema bloquea el intento.
- [ ] El número de camiseta es asignable y editable por el delegado en cualquier momento.
- [ ] No pueden existir dos jugadores con el mismo número de camiseta en el mismo equipo del torneo.
- [ ] Eliminar un jugador del plantel no elimina sus estadísticas del torneo ni de su perfil (soft delete: estado `RETIRADO`).
- [ ] Si el torneo ya inició, se muestra advertencia al eliminar un jugador.

// ─────────────────────────────────────────────────────

private MiembroEquipoDTO toMiembroDTO(JugadorEquipo je, Jugador jugador, EquipoTorneo et) {
    return MiembroEquipoDTO.builder()
            .cedula(je.getCedula())
            .nombre(jugador != null ? jugador.getNombre() : null)
            .posicion(jugador != null ? jugador.getPosicion() : null)
            .piernaHabil(jugador != null ? jugador.getPiernaHabil() : null)
            .alturaCm(jugador != null ? jugador.getAlturaCm() : null)
            .numeroCamiseta(je.getNumeroCamiseta())
            .estado(je.getEstado() != null ? je.getEstado().name() : null)
            .desde(je.getFechaInicio())
            .esDelegado(je.getCedula().equals(et.getDelegadoCedula()))
            .torneoEstado(et.getTorneo().getEstado().name())
            .build();
}

@Transactional
public MiembroEquipoDTO agregarMiembro(String delegadoCedula, UUID equipoTorneoId,
                                        AgregarMiembroRequest req) {
    // 1. Validar que el delegado es dueño de esta inscripción
    var et = equipoTorneoRepo.findById(equipoTorneoId)
            .orElseThrow(() -> new ResourceNotFoundException("Inscripcion no encontrada."));
    if (!et.getDelegadoCedula().equals(delegadoCedula)) {
        throw new RuntimeException("No eres el delegado de esta inscripcion.");
    }

    // 2. Validar que el jugador existe en el padrón
    var jugador = jugadorRepo.findById(req.getCedula())
            .orElseThrow(() -> new ResourceNotFoundException(
                    "Jugador no encontrado. " +
                    "Solicita al administrador que lo cargue en el padron oficial."));

    // 3. Validar que el jugador no tenga membresía ACTIVA en otro equipo del mismo torneo
    if (jugadorEquipoRepo.existsByCedulaAndTorneoIdAndEstado(
            req.getCedula(), et.getTorneo().getId(), EstadoMembresia.ACTIVO)) {
        throw new RuntimeException("El jugador ya pertenece a un equipo en este torneo.");
    }

    // 4. Validar unicidad del número de camiseta (solo entre ACTIVOS)
    if (req.getNumeroCamiseta() != null) {
        if (jugadorEquipoRepo.existsByEquipoTorneoIdAndNumeroCamisetaAndEstado(
                equipoTorneoId, req.getNumeroCamiseta(), EstadoMembresia.ACTIVO)) {
            throw new RuntimeException(
                    "El número " + req.getNumeroCamiseta() + " ya está asignado en este equipo.");
        }
    }

    // 5. Buscar si el jugador ya tuvo una membresía RETIRADA en este torneo
    var existente = jugadorEquipoRepo.findByCedulaAndTorneoId(
            req.getCedula(), et.getTorneo().getId());

    if (existente.isPresent()) {
        // Reactivar: cambiar estado y actualizar datos
        var je = existente.get();
        je.setEquipoTorneo(et);
        je.setEstado(EstadoMembresia.ACTIVO);
        je.setFechaInicio(LocalDate.now());
        je.setFechaFin(null);
        if (req.getNumeroCamiseta() != null) {
            je.setNumeroCamiseta(req.getNumeroCamiseta());
        }
        jugadorEquipoRepo.save(je);
        return toMiembroDTO(je, jugador, et);
    }

    // 6. No existía → crear nueva membresía
    var je = JugadorEquipo.builder()
            .cedula(jugador.getCedula())
            .torneo(et.getTorneo())
            .equipoTorneo(et)
            .fechaInicio(LocalDate.now())
            .estado(EstadoMembresia.ACTIVO)
            .numeroCamiseta(req.getNumeroCamiseta())
            .build();
    jugadorEquipoRepo.save(je);

    return toMiembroDTO(je, jugador, et);
}

@Transactional
public MiembroEquipoDTO removerMiembro(String delegadoCedula, UUID equipoTorneoId, String cedula) {
    var et = equipoTorneoRepo.findById(equipoTorneoId)
            .orElseThrow(() -> new ResourceNotFoundException("Inscripcion no encontrada."));
    if (!et.getDelegadoCedula().equals(delegadoCedula)) {
        throw new RuntimeException("No eres el delegado de esta inscripcion.");
    }

    var je = jugadorEquipoRepo.findByCedulaAndEquipoTorneoId(cedula, equipoTorneoId)
            .orElseThrow(() -> new ResourceNotFoundException("Jugador no encontrado en el plantel."));

    // ⚠️ Proteger al delegado: no puede eliminarse a sí mismo
    if (je.getCedula().equals(et.getDelegadoCedula())) {
        throw new RuntimeException("No puedes eliminar al delegado del plantel.");
    }

    if (je.getEstado() == EstadoMembresia.RETIRADO) {
        return toMiembroDTO(je, jugadorRepo.findById(cedula).orElse(null), et); // idempotente
    }

    // Soft delete: cambiar estado y registrar fecha de salida
    je.setEstado(EstadoMembresia.RETIRADO);
    je.setFechaFin(LocalDate.now());
    // NO se borra el numeroCamiseta — se conserva para estadísticas históricas
    jugadorEquipoRepo.save(je);

    return toMiembroDTO(je, jugadorRepo.findById(cedula).orElse(null), et);
}

@Transactional
public MiembroEquipoDTO actualizarCamiseta(String delegadoCedula, UUID equipoTorneoId,
                                            String cedula, Integer numeroCamiseta) {
    var et = equipoTorneoRepo.findById(equipoTorneoId)
            .orElseThrow(() -> new ResourceNotFoundException("Inscripcion no encontrada."));
    if (!et.getDelegadoCedula().equals(delegadoCedula)) {
        throw new RuntimeException("No eres el delegado de esta inscripcion.");
    }

    var je = jugadorEquipoRepo.findByCedulaAndEquipoTorneoId(cedula, equipoTorneoId)
            .orElseThrow(() -> new ResourceNotFoundException("Jugador no encontrado en el plantel."));

    // Validar unicidad solo entre ACTIVOS (excluyendo al mismo jugador)
    if (numeroCamiseta != null) {
        var ocupado = jugadorEquipoRepo
                .findByEquipoTorneoIdOrderByFechaInicioAsc(equipoTorneoId)
                .stream()
                .filter(j -> j.getEstado() == EstadoMembresia.ACTIVO
                        && j.getNumeroCamiseta() != null
                        && j.getNumeroCamiseta().equals(numeroCamiseta)
                        && !j.getCedula().equals(cedula))
                .findAny();
        if (ocupado.isPresent()) {
            throw new RuntimeException(
                    "El número " + numeroCamiseta + " ya está asignado a otro jugador activo.");
        }
    }

    je.setNumeroCamiseta(numeroCamiseta);
    jugadorEquipoRepo.save(je);

    return toMiembroDTO(je, jugadorRepo.findById(cedula).orElse(null), et);
}

### 4. `DelegadoController.java` — nuevos endpoints

```java
@PostMapping("/equipo-torneo/{equipoTorneoId}/miembros")
public ResponseEntity<MiembroEquipoDTO> agregarMiembro(
        @PathVariable UUID equipoTorneoId,
        @RequestBody @Valid AgregarMiembroRequest req,
        @AuthenticationPrincipal Jwt jwt) {
    return ResponseEntity.ok(
        delegadoService.agregarMiembro(jwt.getClaimAsString("cedula"), equipoTorneoId, req));
}

@PostMapping("/equipo-torneo/{equipoTorneoId}/miembros/{cedula}/remover")
public ResponseEntity<MiembroEquipoDTO> removerMiembro(
        @PathVariable UUID equipoTorneoId,
        @PathVariable String cedula,
        @AuthenticationPrincipal Jwt jwt) {
    return ResponseEntity.ok(
        delegadoService.removerMiembro(jwt.getClaimAsString("cedula"), equipoTorneoId, cedula));
}

@PatchMapping("/equipo-torneo/{equipoTorneoId}/miembros/{cedula}/camiseta")
public ResponseEntity<MiembroEquipoDTO> actualizarCamiseta(
        @PathVariable UUID equipoTorneoId,
        @PathVariable String cedula,
        @RequestBody @Valid ActualizarCamisetaRequest req,
        @AuthenticationPrincipal Jwt jwt) {
    return ResponseEntity.ok(
        delegadoService.actualizarCamiseta(
            jwt.getClaimAsString("cedula"), equipoTorneoId, cedula, req.getNumeroCamiseta()));
}
```

## Errores corregidos del plan original

| # | Error | Solución aplicada |
|---|-------|-------------------|
| 1 | Choque con unique constraint `(cedula, torneo_id)` al re-agregar un `RETIRADO` | `agregarMiembro()` busca si ya existe un registro RETIRADO y lo reactiva (`estado=ACTIVO`, `fechaFin=null`) en vez de insertar uno nuevo |
| 2 | `findByEquipoTorneoId(...)` sin `Order` no existe en el repositorio | `actualizarCamiseta()` ahora usa `findByEquipoTorneoIdOrderByFechaInicioAsc()` que ya existe |
| 3 | `toMiembroDTO(je, jugador, et)` helper no existe | Se creó el helper privado `toMiembroDTO()` y se actualizaron `miembros()` y `miembrosAdmin()` para reusarlo |
| 4 | Falta guardia server-side contra eliminar al delegado | `removerMiembro()` lanza error si `je.getCedula().equals(et.getDelegadoCedula())` |

## Riesgos mitigados

| Riesgo | Solución |
|--------|----------|
| La query de unicidad de camiseta contaba también `RETIRADOS` | `existsByEquipoTorneoIdAndNumeroCamisetaAndEstado(..., ACTIVO)` filtra solo activos |
| `@NotNull` en `ActualizarCamisetaRequest` impedía quitar el número | Se cambió a `Integer nullable` |
| `toMiembroDTO()` repetido inline en `miembros()` y `miembrosAdmin()` | Refactorizado al helper privado |
| Frontend: handlers y estados (`agregando`, `handleAgregarJugador`, etc.) no existían | Se documentaron completos en la sección de frontend |

## Reglas de negocio a implementar

- Un jugador solo puede tener **una membresía ACTIVA** por torneo. La unique constraint `(cedula, torneo_id)` en BD lo refuerza. Si el jugador ya existe como `RETIRADO`, se reactiva en lugar de crear duplicado (ver punto 5 de `agregarMiembro()`).
- El delegado **no** puede eliminarse a sí mismo del plantel: validado tanto en frontend (`!m.esDelegado`) como en backend (`removerMiembro()` lanza error).
- Al eliminar un jugador: soft delete con estado `RETIRADO` + `fecha_fin` — las estadísticas se conservan.
- Un jugador `RETIRADO` puede volver a ser agregado: se busca el registro existente por `cedula + torneoId` y se cambia a `ACTIVO`, actualizando `equipoTorneo`, `fechaInicio` y opcionalmente `numeroCamiseta`.
- El número de camiseta es único entre miembros **ACTIVOS** del mismo equipo en el torneo. Los `RETIRADOS` no bloquean el número.
